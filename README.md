# Ottawa Tree Viewer

Static map viewer for Ottawa trees using PMTiles + MapLibre.

## Workspace

- `apps/geojson-builder`: transforms local GeoJSON, builds PMTiles and metadata.
- `apps/viewer`: static React + Vite viewer.
- `packages/shared`: shared utilities and types.

## Local data policy

Raw GeoJSON is user-supplied and not committed.

### Data source

Download the City of Ottawa **Tree Inventory** from [open.ottawa.ca](https://open.ottawa.ca/datasets/tree-inventory/explore) and use it as described below.

**Licence:** Contains information licensed under the [Open Government Licence – City of Ottawa](https://ottawa.ca/en/city-hall/open-transparent-and-accountable-government/open-data/open-data-licence-version-20).

1. Download the tree GeoJSON and place it at `data/raw/trees.geojson` (or pass a custom path).
2. Build data:

```bash
pnpm run build:data
```

3. Build viewer:

```bash
pnpm run build:viewer
```

`apps/viewer/public/trees.pmtiles` and metadata JSON are generated by the builder.

## Deploying (Cloudflare Pages and similar)

PMTiles relies on **HTTP Range** requests (partial content, **206**). Static hosts that do not honor ranges may download the entire archive for every tile request. For production builds, set **`VITE_PMTILES_URL`** to a full URL where the file is served with Range support (for example jsDelivr or raw GitHub for a public repo, or object storage). Vite inlines this at build time, so define it in your CI/hosting environment variables. See `apps/viewer/.env.example`.

If `VITE_PMTILES_URL` is unset, the app uses same-origin `/trees.pmtiles` (fine for local development with `pnpm dev`).

After deploy, confirm in the browser **Network** tab that requests for `trees.pmtiles` return **206** and small byte ranges when you pan and zoom.
